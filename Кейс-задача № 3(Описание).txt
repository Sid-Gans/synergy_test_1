Подробное описание решения


Анализ предметной области:
  Определены основные сущности: клиенты, услуги, страны, отели, заказы
  Выделены справочники (статические данные) и оперативные данные (заказы)
Нормализация базы данных:
  Все таблицы приведены к 3НФ (третьей нормальной форме)
  Исключена избыточность данных за счет связей между таблицами
Определение связей:
  Один-ко-многим между Countries и Hotels
  Один-ко-многим между Clients и Orders
  Один-ко-многим между Services и Orders
  Один-ко-многим между Hotels и Orders
Класс TourismDB:
  Инкапсулирует логику работы с базой данных
  Предоставляет методы для выполнения запросов и получения данных
  Обрабатывает ошибки подключения и выполнения запросов
Методы класса:
  execute_query() - для выполнения INSERT, UPDATE, DELETE
  fetch_data() - для выполнения SELECT запросов
  close() - для закрытия соединения
Пример использования:
  Демонстрация CRUD-операций (Create, Read, Update, Delete)
  Показ сложного запроса с JOIN для получения связанных данных
Безопасность:
  Использование параметризованных запросов для защиты от SQL-инъекций
  Обработка ошибок при работе с базой данных
Масштабируемость:
  Легко добавлять новые таблицы и связи
  Возможность расширения функционала класса TourismDB
  Производительность:
  Индексы для часто используемых полей
  Проверки (CHECK) для обеспечения целостности данных
